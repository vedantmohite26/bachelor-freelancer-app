## 2026-06-25 - Virtualization and Stream Caching in Ratings Module
**Learning:** `HelperReviewsScreen` used the `shrinkWrap: true` and `NeverScrollableScrollPhysics` anti-patterns within a `SingleChildScrollView`, which disables Flutter's UI virtualization. This causes O(N) widget building and memory usage for large lists. Additionally, `RatingSummaryCard` was triggering a new Firestore `Future` on every build, and `HelperReviewsScreen` was creating a new `Stream` on every build.
**Action:** Replace `SingleChildScrollView` + `Column` + `ListView.builder(shrinkWrap: true)` with `CustomScrollView` and `SliverList` to enable O(visible) virtualization. Cache `Stream` and `Future` objects in `StatefulWidget`'s `initState` to prevent redundant network requests and improve UI stability.
